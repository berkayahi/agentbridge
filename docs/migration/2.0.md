# AgentBridge 2.0 migration

The v2 SQLite bootstrap is explicit. A fresh database is created through the
v2 migration ledger; a legacy database returns `migration required` and is not
mutated by daemon startup. Run the dedicated migration command only after a
verified backup and with no active daemon:

```sh
agentbridge migrate --database /absolute/path/agentbridge.db
agentbridge doctor --database /absolute/path/agentbridge.db --json
```

Mode changes are durable and reject active work. Managed activation additionally
requires a matching enrolled device key, record, trust epoch, and replay state.
The backup manifest carries only public identity facts and can never restore a
private key or lower an accepted trust floor.
